Interaction Flow

  • Avery googles "astronomy event albuquerque".
  • Scrolls through and clicks the link to our site.
  • Our server returns the splash page.
  • Avery sees a carousel with today's astronomy events, he clicks the arrow to navigate more quickly through carousel.
  • Server is chillin' until....
  • Avery clicks an event he wants to get more info about.
  • Server returns automatic prompt to create account/sign in.
  • Avery creates his account because this website is sweet.
  • Server stores Avery's account information in user table and returns the event page.
  • Avery can now view specific event details and comment directly to event with questions.
  • Avery has a question! So he posts a comment in the event discussion and checks the box to be updated when changes have been made to the event/someone comments back.
  • Server stores comment in comment table and adds comment to UI.
  • Avery sees comment closes website on work desktop to wait for a response as he drives home.
  • User Amanda comments back about event question.
  • Server stores comment in comment table, adds comment to UI and sends email alert to Avery's user email.
  • As Avery is driving home he receives an email update alert and opens it (while being stopped at a light). He clicks link in email and navigates directly to thread in safari on his phone.
  • Server media queries the screen size and returns the event post and comments.
  • Avery reads when the star party will start and now has a cool astronomy event to attend with his kid.
